LeetCode：#1 Two Sum

2016-10-08

题目与样例

原题地址：LeetCode 1.Two Sum

Given an array of integers, return indices of the two numbers such that they add up to a specific target.

You may assume that each input would have exactly one solution.

中文大意：给一个数组和一个数，从数组中找出两个数之和等于这个给的数，返回数组中这两个数的索引。

Example：
Given nums = [2, 7, 11, 15], target = 9,

Because nums[0] + nums[1] = 2 + 7 = 9,
return [0, 1].

解法1：暴力法

题目已经很明白了，我们用两次循环来对每每两个数求和来判断与给定数target比较，相同则返回索引即可，这是最容易想的一种方法。

这种方法的时间复杂度为$O(n^2)$，空间复杂度为$O(1)$。

解法2：两次循环+哈希表

对于刚才的方法，我们是在有一个数的基础上，遍历数组去找另一个数，查找的效率是$O(n)$，对此我们可以从查找效率上想办法进行优化下，我们知道哈希表的查找效率是$O(1)$，那么就用哈希表来进行查找。

先用一个循环来构建一个哈希表，在C11标准中，推荐使用unordered_map来代替hash_map。表的key值为数组中的数，val值为索引号，这样我查到了对应的key就能知道它在数组中的位置。

这种方法的时间复杂度为$O(n)$，因为查找的复杂度是$O(1)$,空间复杂度是$O(n)$因为构建了一张哈希表。

解法3：一次循环+哈希表

我们可以把建表的过程放进查找的过程中，在具体实现时，我们可以考虑一些小技巧，比如在上种方法中判断中有两个条件，其中第二个条件是为了防止出现自己加上自己刚好等于结果的情况发生。我们在一边建表一边查询的方法中，如果把建表过程放在查询后，即查询不到自己，就能避免自己加自己的情况发生，代码也显得更加简练。

时间复杂度和空间复杂度如解法2一样，都是$O(n)$。
